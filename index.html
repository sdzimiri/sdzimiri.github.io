AWS Elastic File System

Posted on 2021-08-16 Symbols count in article: 5.3k Reading time ≈ 5 mins.

Introduction

The Amazon EFS and Amazon FSx services provide easily deployable file systems for use with Linux and Windows instances.

Amazon EFS

Amazon EFS is based on the Network File System (NFS) protocol that Linux-based systems use. Amazon EFS provides a simple, serverless, set-and-forget, elastic file system that allows you to share file data without provisioning or managing storage. It can be used with AWS Cloud services and on-premises resources and is built to scale on demand to petabytes without disrupting applications. Using Amazon EFS, you can grow or shrink your file systems automatically as needed, eliminating the need to provision and manage capacity to accommodate growth.

Introduction

Block storage devices divide their underlying media into individual units called blocks. These blocks are used to store data. Each block is assigned a unique address so that the block and its data can be read or written to the storage device. Blocks represent the smallest unit of I/O that a storage device can perform, and the speed with which blocks can be processed contributes to storage performance characteristics such as IOPS and throughput. This lesson describes two types of AWS block storage solutions: an instance store and Amazon EBS.

Introduction (S3)

Amazon Simple Storage Service (Amazon S3) is storage for the Internet. It is designed to make web-scale computing easier.

Amazon S3 has a simple web services interface that you can use to store and retrieve any amount of data, at any time, from anywhere on the web. It gives any developer access to the same highly scalable, reliable, fast, and inexpensive data storage infrastructure that Amazon uses to run its own global network of web sites.

Introduction

Amazon EMR is a web service that enables businesses, researchers, data analysts, and developers to easily and cost-effectively process vast amounts of data. EMR utilizes a hosted Hadoop framework running on Amazon EC2 and Amazon S3. Managed Hadoop framework for processing huge amounts of data. It also support Apache Spark, HBase, Presto and Flink. Amazon EMR is most commonly used for log analysis, financial analysis, or extract, translate and loading (ETL) activities.
