Introduction to AWS storage services

Posted on 2021-08-16 Edited on 2021-09-08 In AWS Certification , AWS CCP , AWS SAA Symbols count in article: 2.2k Reading time ≈ 2 mins.

Introduction

AWS provides low-cost data storage with high durability and availability. More specifically, AWS offers storage choices for backup, archiving, and disaster recovery use cases and provides block, file, and object storage. This lesson provides an overview of the main storage concepts.

Block storage

Block storage is a type of storage that emulates the behavior of regular hard disks for data storage. You can use it to store structured data such as databases. AWS offers two options for block storage. You can easily scale up your block storage resources by increasing the volume size or adding more volumes. Depending on the durability that you are aiming for, you can choose between an Amazon Elastic Compute Cloud (Amazon EC2) instance store for ephemeral block storage or Amazon Elastic Block Store (Amazon EBS) for persistent block storage.

File storage

File storage is a method for storing unstructured data in the cloud that provides servers and applications shared access to data. This capability makes cloud file storage ideal for workloads that rely on shared file systems. Using file systems, users can create, delete, modify, read, and write files using a hierarchical storage system that they can logically organize into directory trees for intuitive access.

You can use Amazon Elastic File System (Amazon EFS) for file storage in the cloud. Amazon EFS is based on the Network File System (NFS) protocol that Linux-based systems use. For Microsoft Windows-based workloads, you can leverage Amazon FSx file storage, a fully managed file system based on the Server Message Block (SMB) protocol. Additionally, the Amazon FSx for Lustre file system provides consistent, low-latency storage for compute-intensive Linux workloads.

Object storage

Object storage provides massively scalable, cost-effective storage to store any type of data in its native format. AWS object storage solutions Amazon S3 and Amazon S3 Glacier offer very high availability, durability, and scalability while providing security and compliance. AWS object storage solutions are well suited for a large range of use cases, such as big data analytics, data archiving and compliance, backup, and disaster recovery.

Hybrid storage

Many customers want to take advantage of the benefits of cloud storage, but they have applications running on premises that require low-latency access to their data or need rapid data transfer to the cloud. AWS hybrid cloud storage architectures allow you to connect on-premises applications and systems to cloud storage or to migrate your data to the AWS Cloud.
