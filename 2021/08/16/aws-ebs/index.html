AWS Elastic Block Storage

Posted on 2021-08-16 In AWS Certification , AWS CCP , AWS SAA Symbols count in article: 4.3k Reading time ≈ 4 mins.

Introduction

Block storage devices divide their underlying media into individual units called blocks. These blocks are used to store data. Each block is assigned a unique address so that the block and its data can be read or written to the storage device. Blocks represent the smallest unit of I/O that a storage device can perform, and the speed with which blocks can be processed contributes to storage performance characteristics such as IOPS and throughput. This lesson describes two types of AWS block storage solutions: an instance store and Amazon EBS.

Instance store

An instance store provides temporary block-level storage for your instance. This storage is located on disks that are physically attached to the host computer. An instance store is ideal for temporary storage of information that changes frequently, such as buffers, caches, scratch data, and other temporary content. It is also ideal for data that is replicated across a fleet of instances, such as a load-balanced pool of web servers.

An instance store consists of one or more instance store volumes exposed as block devices. The size of an instance store and the number of devices available vary by instance type. You can specify instance store volumes for an instance only when you launch it. You can’t detach an instance store volume from an instance.

The data in an instance store persists only during the lifetime of its associated instance. If an instance reboots (intentionally or unintentionally), data in the instance store persists. However, data in the instance store is lost under any of the following circumstances:

  • The underlying disk drive fails
  • The instance stops
  • The instance hibernates
  • The instance terminates

Instance store volumes

The instance type determines the size of the instance store available and the type of hardware used for the instance store volumes. Instance store volumes are included as part of the instance’s usage cost. You must specify the instance store volumes that you’d like to use when you launch the instance. Then, you need to format and mount the instance store volumes before using them.

For enhanced performance, some instance types use Non-Volatile Memory Express (NVMe) or SATA-based solid state drives as the instance store to deliver very low latency with high Input/Output operations per second (IOPS).

Amazon EBS

Amazon EBS is an easy-to-use, high-performance, block-storage service designed for use with Amazon EC2 for both throughput and transaction-intensive workloads at any scale. Amazon EBS is suited for a broad range of workloads, such as relational and nonrelational databases, enterprise applications, containerized applications, big data analytics engines, file systems, and media workflows.

Benefits of Amazon EBS

Amazon EBS is highly durable and available. Amazon EBS volumes are replicated within an Availability Zone and can easily scale to petabytes of data. You can also use EBS snapshots with automated lifecycle policies to back up your volumes in Amazon S3 while ensuring geographic protection of your data and business continuity. A subsequent lesson will discuss Amazon S3.

Amazon EBS offers different volume types to balance optimal price and performance. You can achieve single-digit millisecond latency for high-performance database workloads such as SAP HANA or gigabyte per second throughput for large, sequential workloads such as Hadoop. You can change volume types, tune performance, or increase volume size without disrupting your critical applications so that you have cost-effective storage when you need it.

EBS volume types

Amazon EBS volume types differ in performance characteristics and price so that you can tailor your storage performance and cost to the needs of your applications. The following are the volume types available:

Solid state drive (SSD) volumes are optimized for transactional workloads involving frequent read/write operations with small I/O size where the dominant performance attribute is IOPS.

  • General Purpose SSD volumes provide a balance of performance and costs and are recommended for most workloads. General Purpose SSD volumes offer the possibility to burst to higher I/O performances using I/O credits. I/O credits accumulate when the volume stays below the baseline performance level (defined by the volume size). When needed, I/O credits can be used to burst to higher performances until the number of credits returns to zero.

  • Provisioned IOPS SSD volumes provide high performance for mission-critical, low-latency, or high-throughput workloads.

Hard disk drive (HDD) volumes are optimized for large streaming workloads where the dominant performance attribute is throughput.

  • Throughput Optimized HDD volumes are a low-cost HDD designed for frequently accessed, throughput-intensive workloads. Like the General Purpose SSD volume types, Throughput Optimized HDD can burst to higher performance levels using I/O credits.
  • Cold HDD volumes are the lowest-cost HDD designed for less frequently accessed workloads. In Cold HDD volumes, you can choose sc1 volumes if you need burstable performances.
