What is a computer network?

A computer network is a collection of computing devices that are physically and logically connected to communicate and share resources.

A node refers to any devices that are on the network, whereas the host refers to the servers on the network. It is a device to which other computers can connect to access data or other resources.

Switch

A switch is a small hardware device that connects multiple devices in the same network. It directs data traffic from a source to a destination device by switching the data from the source to its destination.

Computing devices connect to a switch by using a network interface card (NIC) and a cable.

Router

A router acts as a dispatcher as it decides which way to send each packet of data. A router is located at any gateway (where the network meets another). A router connects multiple switches, and their respective networks, to enable communication between different networks.

Types of computer networks

Two of the most common types of computer networks are local area networks (LANs) and wide-area networks (WANs).

Local area network (LAN)

A LAN connects devices in a limited geographical area such as a floor, building, or campus.LANs commonly use the Ethernet standard for connecting devices and usually have a high data transfer rate. Wireless technology is also commonly used for a LAN.

Wide area network (WAN)

A WAN connects devices in a large geographical area such as in multiple cities or countries. WANs are used to connect LANs and use technologies such as fiber-optic cables and satellites to transmit data. The internet is considered to be the largest WAN.

Client-Server model

In a client-server network model, the data management and application hosting are centralized at the server and distributed to the clients. All clients on the network must use the designated server to access shared files and information that are stored on the serving computer.

If the server goes down, no client is able to access the network until the server is restored. Examples of client-server models are:

  • File server and desktop clients
  • Print server and desktop clients

Peer-to-peer model

In a peer-to-peer network model, each node has its own data and applications, and is responsible for its own management and security.

The peer-to-peer model is a distributed architecture that shares tasks or workloads among peers. Peers are equally privileged participants in the architecture. Files can be shared directly between the systems on the network without the need for a central server.

This model might be considered under the following conditions:

  • Users are responsible for backing up each node.
  • Security requirements are not restrictive
  • A limited number of peers are used.

OSI Model

The OSI Model (Open Systems Interconnection Model) is a conceptual framework used to describe the functions of a networking system. Each layer serves the above it and depends on the layer below it

Static Routing

In static routing, a network administator specifies the exact route the traffic will take. A treasure map often shows a very specific path to the treasure. Following the same exact path each time to get somewhere is a kind of static routing. This is like an end-to-end static route, where each node in the network has a static route to the next node in the path to the final destination.

Static routing advantages:

  • Maximum control: because static routes are manually added, they provide predictable route prioritization.
  • Simple to configure on small networks

Static routing disadvantages:

  • It is difficult to manage at scale.
  • Manual administration is slow and subject to errors.
  • The network cannot spend to changing conditions easily.

Dynamic Routing

In computer network, dynamic routing is automatic routing determined by the network devices themselves. In dynamic routing, the routers know what networks are directly connected to them. A major aspect of dynamic routing is that it enables the network to adapt to changing conditions, such as failed links, by adjusting routes as necessary.

Dynamic routing advantages:

  • Scales well as the network grows.
  • Provides fewer opportunities for human error.
  • Adapts to changing conditions.

Dynamic routing disadvantages:

  • Less control
  • Emergent routing

Address Resolution Protocol (ARP)

Imagine that John lives in a new housing development where houses are built on only one street so far. He is new to the neighborhood and doesn’t know anyone yet. He goes for a walk and finds a book that is lying in the street. It says “property of Jane.” He would like to return the book to Jane, but he doesn’t know which house Jane lives in. How can he find out? One way is for John to write a note that says that he has Jane’s book and can deliver it. In the note, he provides his contact information, including the house that he lives in because everyone in the neighborhood is friendly. He then places copies of this note in everyone’s mailbox. When Jane reads the note, she contacts John and lets him know which house she lives in so that he can deliver the book to her. Now everyone in the neighborhood knows where John lives and John knows where Jane lives. This scenario is essentially what happens when a computer (John) tries to send an IP packet (the book) to another system (Jane). It is sending the packet on the same local network (the neighborhood) for the first time. It must map the IP address (the name Jane) to the hardware address (the house number). The protocol that governs this process for IPv4 is the ARP.

ARP is part of the TCP/IP protocol suite. It is used to translate IP addresses to hardware addresses, also called MAC addresses. When a network node that runs TCP/IP must know which node on an Ethernet network uses a particular IPv4 address, it broadcasts an ARP request. The broadcast basically says: “Attention! Which node is using IP address xxx.xxx.xxx.xxx?”

The node that owns the specific IPv4 address responds with its own MAC address. The node that sent the request then adds that MAC address to its own ARP table.

For IPv6, the Neighbor Discovery Protocol (NDP) is used for address resolution.

Port Forwading

A port is a way to route traffic to a specific application on a host. This is a virtual point where the network connection start and end. Port forwarding is a function that a border router (or firewall) performs. Its purpose is to direct traffic that comes into a specific port on its public interface to an IP host in the private network. In this way, services in the private network become available to the public network. The publicly advertised port number can be mapped to a different port number on the private network host.

Port Forwading Example

For example, a web server in the private network runs on an IP host with the private IP address 172.16.30.1 on TCP port 8080. Web clients in the internet should be able to access the web server by using a standard HTTP URL. To ensure that they can, the server is advertised to the internet as the public IP address of the router (54.151.109.145) that runs on port 80. Port forwarding that is configured on the router directs incoming traffic on its public interface on port 80. It directs the traffic to the web server (172.16.30.1) on port 8080. Return traffic from the server leaves the private network (through the router) with a source address and port number of the router (54.151.109.145:80).

Dynamic Host Configuration Protocol

Before 1993, IPv4 addresses—including subnet masks, gateways, and other parameters—needed to be configured manually and individually by network administrators. As organization networks became larger, the risk of configuration errors increased, and this caused a heavy maintenance workload. A protocol was needed urgently to automate the IP configuration process in large networks. To solve those issues, a new protocol was released: DHCP.

DHCP enables automatic IP address configuration for almost all devices by exchanging messages between devices and a DHCP server. DHCP can also automatically configure other parameters such as a default gateway address, a DNS server IP address, or a subnet mask.

DHCP uses lease time to dynamically assign IP addresses. When an IP address is no longer used by a device, it goes back to the pool of IP addresses. The IP address can then be assigned to another device, if needed. This process enables you to connect more devices to the network than the total number of IP addresses that are available in the DHCP server.

Advantages of DHCP

  • In addition to providing the first fully automated solution to assign IP addresses, DHCP offers increased flexibility and scaling.
  • Only the devices that are currently in use get an IP address, which enables you to have more devices than IP addresses on a network.
  • The modifications of the parameters that are made on a DHCP server are centralized in the DHCP server. The modifications are then automatically propagated to the devices when they are rebooted.

Domain Name System (DNS)

DNS is a distributed services that’s used to translate the domain names of devices that are connected to the internet or a private network into IP addresses. A domain name is easier to remember than a numerical IP address of a host. It is possible to attach other information to a domain name, in addition to an IP address. For example, different records types can be used to add more security for data transmission, to avoid spam, or to associate email addresses with phone numbers.

DNS Hierarchy

The DNS hierarchy is comprised of the following elements:

  • Root Level
  • Top Level Domains
  • Second Level Domains
  • Sub-Domain
  • Host

DNS Record Types

HOST

Host records correspond too the A and AAAA records. These RRs store IP addresses for DNS names. The A record is used to map a domain name to an IPv4 address. The AAAA record is an extension of the A record, which enables mapping a domain name to an IPv6 address. For example, when a client sends a request asking for the IP address of a domain name, the request goes to its DNS server. The server then looks for an A (or AAAA) record that corresponds to that domain name. After, the server returns the IP address to the client.

CNAME

This record is called canonical name (CNAME). It’s used to map a domain name to another domain name. The first domain name is called the alias and the other domain name is referred to as the canonical name. CNAME records always point to the domain namen and never point to IP addresses. This record can be useful if several services run at the same time on several ports, but use only one IP address.

MX

The MX record stands for Mail Exchange. It identifies mail servers, which are also known as message transfer agents, for a given DNS name. If an MX record is present for the DNS name example.com, at least one mail server must process and accept email messages for that domain. When a client wants to send an email message to a domain name, a mail server sends a query to the DNS server of the target domain name. The mail server asks for the MX record. The DNS server will send back the MX record as a response so that the client knows what mail server to use. When the mail host is selected, the client mail server will initiate a Simple Mail Transfer Protocol (SMTP) connection, beginning with the mail host that has the lowest priority. A single domain can have several configured MX records. To choose between the different mail servers of the MX records, a priority feature is specified, as shown in the following example. A sending agent always starts using the host that has the lowest priority value before it chooses another host, which has the second-lowest value. You can learn more about priorities in the RFC 5321, which is the standard that describes SMTP.

NS

The NS record, which stands for Name Server record, enables the creation of a zone delegation. For a given domain, the NS record directs to another authoritative name server for the domain name.

Domain names often, if not always, have several configured NS records. If one DNS server goes down, another one can take control and give access to the domain name.

When a DNS record is resolved by a DNS server, the DNS server reaches one of the associated name servers. It keeps the results in its cache for a given period of time, which is defined by the TTL parameter. If a DNS record is changed and the TTL hasn’t expired, the DNS server doesn’t know about those changes before the cache expires. To correctly propagate the latest DNS record, the TTL must expire, or another name server must be contacted.

SOA

The SOA record stands for Start of Authority. In DNS, each zone has only one SOA record. In addition to the usual record fields, the SOA record provides specific information that will be detailed later.

First, the SOA record contains information that’s related to the primary name server of the zone. This primary name server is the server with the highest authority for the zone and it has full rights to that zone (read, write, and copy).

The SOA record also provides the email address of the network administrator who’s in charge of the domain name.

Next in the SOA record, you will find the serial number of the zone. This serial number is incremented each time a modification is made to the zone. This serial number is used for zone copy. If a DNS server sees that two copies of the same zone have different serial numbers, it knows that the copy with the lowest number must be updated.

Finally, a refresh time for the zone is present in the SOA record. This refresh time is used by secondary DNS servers to know when to check for updates. The refresh time can be changed, depending on how often a zone will be modified. The retry interval is used if a previous updated failed, and it specifies how long a secondary DNS server should wait before it retries updating the zone. Finally, the expire timer corresponds to the time that a secondary DNS server should consider that the zone is no longer available. By convention, the expire time should be larger than the sum of the refresh and retry intervals.

PTR

The PTR record, or Pointer record, provides a mapping between an IP address and a name. This record can be seen as the exact opposite of an A or AAAA record. This record means that given an IP address, the domain name that corresponds to that IP address can be looked up. This process is called reverse lookup and it will be discussed in more detail in another lesson. This system is useful for finding which devices correspond to IP addresses in a log file.
